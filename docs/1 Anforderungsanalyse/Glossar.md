# Begriffe
Generell versuchen wir englische begriffe für sowas zu verwenden. Ausnahmen darf es geben wenn der Englische Begriff eher unbekannt ist

| Begriff          | Beschreibung                                                                                                                                                                                                                                                                                                              |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Action           | atomare Operation, die von einem Akteur initiiert wird (z. B. Karte ziehen).                                                                                                                                                                                                                                              |
| Command          | Ein Objekt, das eine konkrete Spielaktion (z. B. "Karte ausspielen") kapselt, um diese auszuführen oder rückgängig zu machen (Undo).                                                                                                                                                                                      |
| Card             | Ein abstraktes Element im Spiel. Jede Karte besitzt spezifische Attribute (z. B. Wert, Farbe)                                                                                                                                                                                                                             |
| Deck             | geordnete oder ungeordnete Sammlung von Card-Objekten. Das Framework stellt Fabrikmethoden zur Generierung verschiedener Decks bereit.                                                                                                                                                                                    |
| Framework/Engine | Der generische Kern des Systems, das den generellen Ablauf (z. B. GameLoop) vorgibt und spielspezifischen Code (Showcase) durch definierte Schnittstellen aufruft (Inversion of Control). Die Engine steuert den Rundenablauf, verwaltet den Zustand und bietet Schnittstellen für spielspezifische Implementierungen an. |
| GameLoop         | zentrale Kontrollmechanismus des Frameworks, der die Phasen des Spiels sequenziell durchläuft                                                                                                                                                                                                                             |
| GameState        | aktuelle Zustand des Spiels, der durch das _State Pattern_ verwaltet wird                                                                                                                                                                                                                                                 |
| Hand             | Sammlung aller Karten die sich im besitz des Spielers befinden                                                                                                                                                                                                                                                            |
| Round            | Definiert den Abschnitt das jeder Spieler einmal die Gelegenheit hatte einen Spielzug durchzuführen                                                                                                                                                                                                                       |
| Phase            | Definierter Abschnitt innerhalb eines Spielzugs (z. B. Ziehphase, Spielphase, Schhlagphase, Dazulegphase, Prüfphase). Die Zustände werden durch das _State Pattern_ gesteuert.                                                                                                                                            |
| Turn             | Ein abgeschlossener Spielabschnitt eines Akteurs, der aus mehreren Phasen bestehen kann.                                                                                                                                                                                                                                  |
| Table            | Tisch bezeichnet alle für die Spieler sichtbaren Karten                                                                                                                                                                                                                                                                   |
| Undo             | Die Möglichkeit, die letzte Aktion des Spielers durch das _Command Pattern_ rückgängig zu machen                                                                                                                                                                                                                          |
| Player           | bezeichnet jeden Spieler ob menschlich oder algorithmus                                                                                                                                                                                                                                                                   |
| Computer         | bezeichnet einen nicht-menschlichen Spieler                                                                                                                                                                                                                                                                               |
| User             | bezeichnet einen Benutzer des Spiels also den menschlichen Spieler                                                                                                                                                                                                                                                        |
| Win Condition    | Bezeichnet eine Kondition die unter Erfüllung den Spieler gewinnen lässt und immer wieder vom System geprüft wird. Bspw. keine Handkarten mehr                                                                                                                                                                            |
| Winner           | Spieler der Win Condition erreicht hat                                                                                                                                                                                                                                                                                    |
# Pattern

| Pattern          | Bschreibung                                                                                                                                                                                                                                 |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Command Pattern  | Pattern zur Kapselung einer Anfrage als Objekt, wodurch Parameterisierung von Klienten mit verschiedenen Anfragen, Warteschlangen oder Protokollierung von Anfragen und Unterstützung von rückgängig machbaren Operationen ermöglicht wird. |
| Factory Method   | Pattern, das eine Schnittstelle zum Erzeugen eines Objekts definiert, aber Unterklassen die Entscheidung überlässt, welche Klasse instanziiert wird. Hier zur Generierung von Kartendecks genutzt.                                          |
| State Pattern    | Pattern, das es einem Objekt ermöglicht, sein Verhalten zu ändern, wenn sich sein interner Zustand ändert                                                                                                                                   |
| Strategy Pattern | Pattern, das eine Familie von Algorithmen definiert, sie kapselt und sie austauschbar macht. Hier für variable Siegbedingungen oder Karteneffekte eingesetzt.                                                                               |
